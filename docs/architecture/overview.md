## レイヤー構成

### 1. プレゼンテーション層

クライアントからのリクエストを処理するレイヤーです。

- **REST API**: Django REST Frameworkベース
- **GraphQL**: Graphene-Djangoベース
- **WebSocket**: Django Channelsベース

### 2. アプリケーション層

ビジネスロジックを処理するレイヤーです。

- **ViewSets**: リクエスト処理とレスポンス生成
- **Serializers**: データ変換とバリデーション
- **Filters**: クエリフィルタリング

### 3. ドメイン層

ビジネスエンティティを表現するレイヤーです。

- **Models**: Djangoモデル
- **Signals**: イベント通知

### 4. インフラストラクチャ層

外部システムとの連携を担当するレイヤーです。

- **Database**: データ永続化
- **Storage**: ファイルストレージ
- **Cache**: キャッシュシステム

## データフロー

### REST APIリクエスト

```
1. クライアント → HTTPリクエスト
2. URLRouter → ViewSet選択
3. ViewSet → 認証・権限チェック
4. Filter → クエリフィルタリング
5. Serializer → データ変換
6. Response → クライアント
```

### GraphQLリクエスト

```
1. クライアント → GraphQLクエリ
2. Schema → Query/Mutation選択
3. Resolver → データ取得
4. Type → データ変換
5. Response → クライアント
```
